# Squid CLI

The Squid CLI is a powerful command-line tool for scaffolding, managing, and deploying Squid indexers. It streamlines the development workflow from project creation to production deployment.

## Installation

```bash
npm install -g @subsquid/cli
```

<Check>
Verify installation with `sqd --version`
</Check>

## Core Commands

### Project Management

#### Initialize New Project
```bash
# Create from template
sqd init my-indexer -t evm

# Create from GitHub repository
sqd init my-indexer -t https://github.com/subsquid-labs/showcase01-all-usdc-transfers

# List available templates
sqd init --help
```

#### Project Development
```bash
# Install dependencies
sqd install

# Build project
sqd build

# Generate TypeORM models
sqd codegen

# Apply database migrations
sqd migration:apply

# Start development server
sqd serve:dev
```

### Process Management

#### Run Processor
```bash
# Start processor
sqd process

# Run with specific configuration
sqd process:prod
```

#### Database Operations
```bash
# Create migration
sqd migration:create

# Apply migrations
sqd migration:apply

# Revert migration
sqd migration:revert
```

### Deployment

#### SQD Cloud Deployment
```bash
# Deploy to SQD Cloud
sqd deploy

# Deploy with custom name
sqd deploy --name my-production-indexer

# Check deployment status
sqd deploy:status
```

## Configuration

### squid.yaml
The main configuration file for your Squid project:

```yaml
manifestVersion: subsquid.io/v0.1
name: my-indexer
version: 1
description: My awesome indexer

build:
  dockerfile: Dockerfile

deploy:
  addons:
    postgres:
      version: 14
  processor:
    cmd: ["node", "lib/main.js"]
  api:
    cmd: ["npx", "squid-graphql-server"]
```

### Environment Variables
```bash
# Database connection
DB_NAME=squid
DB_PORT=5432
DB_HOST=localhost
DB_USER=postgres
DB_PASS=postgres

# SQD Cloud
SQD_TOKEN=your-token-here
```

```bash

sqd init my-project -t https://github.com/your-org/custom-template

# Use local directory as template
sqd init my-project -t ./local-template
```

## Advanced Usage

### Custom Commands
Define custom commands in `commands.json`:

```json
{
  "commands": {
    "build:prod": {
      "description": "Build for production",
      "cmd": ["npm", "run", "build:prod"]
    },
    "test:e2e": {
      "description": "Run end-to-end tests",
      "cmd": ["npm", "run", "test:e2e"]
    }
  }
}
```

### Multi-environment Setup
```bash
# Development
sqd process --env development

# Staging
sqd process --env staging

# Production
sqd process --env production
```

## Troubleshooting

### Common Issues

<Accordion>
  <AccordionGroup>
    <Accordion title="Permission denied errors">
      Ensure you have proper permissions and are logged in to SQD Cloud:
      ```bash
      sqd auth
      ```
    </Accordion>
    
    <Accordion title="Build failures">
      Clear cache and rebuild:
      ```bash
      sqd clean
      sqd build
      ```
    </Accordion>
    
    <Accordion title="Database connection issues">
      Check your environment variables and database status:
      ```bash
      sqd db:status
      ```
    </Accordion>
  </AccordionGroup>
</Accordion>

### Debug Mode
```bash
# Enable verbose logging
sqd --verbose process

# Debug specific command
DEBUG=* sqd build
```

## Best Practices

- Use version control for your Squid projects
- Test locally before deploying to SQD Cloud
- Use environment-specific configurations
- Regularly update CLI to latest version
- Monitor deployment logs for issues

## CLI Reference

### Global Options
- `--help, -h` - Show help
- `--version, -V` - Show version
- `--verbose, -v` - Verbose output
- `--quiet, -q` - Suppress output

### Command Categories
- **Project**: `init`, `build`, `clean`
- **Database**: `migration:*`, `codegen`
- **Development**: `serve:*`, `process`
- **Deployment**: `deploy`, `auth`
- **Utilities**: `explorer`, `ls`

For complete command reference, run:
```bash
sqd --help
```
