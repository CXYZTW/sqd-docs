---
title: "CLI cheatsheet"
description: "This guide provides a quick reference to the commands needed to launch the squid and manage the database."
---

### Install dependencies[​](#install-dependencies "Direct link to Install dependencies")

```
npm i
```

### Compile the project[​](#compile-the-project "Direct link to Compile the project")

```
npx tsc
```

### Launch Postgres database to store the data[​](#launch-postgres-database-to-store-the-data "Direct link to Launch Postgres database to store the data")

```
docker compose up -d
```

### Apply database migrations to create the target schema[​](#apply-database-migrations-to-create-the-target-schema "Direct link to Apply database migrations to create the target schema")

```
npx squid-typeorm-migration apply
```

### Run indexer[​](#run-indexer "Direct link to Run indexer")

```
node -r dotenv/config lib/main.js
```

### Check out the indexed swaps[​](#check-out-the-indexed-swaps "Direct link to Check out the indexed swaps")

```
docker exec "$(basename "$(pwd)")-db-1" psql -U postgres \  -c "SELECT id, logs_count, found_at FROM contract ORDER BY logs_count desc LIMIT 10"
```

You can use the `sqd` utility to shorten these commands and manage their interrelations. [Learn more here](/squid-cli/commands-json/).
